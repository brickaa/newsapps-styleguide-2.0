Utils

The grid @mixins are located in helpers/_utils.scss. The News Apps graphic and app kits include the Bourbon Neat semantic grid framework.

To include an element in a responsive grid design, add the class .container to the parent-most HTML element. We've overriden Bourbon's $max-width to 1081px, which is typical for our app projects. To change how wide the grid can stretch, set the variable $max-width in your project.

Breakpoints

For ease of use, we've created two types of default breakpoints — both of which you can override to best fit your project or not use at all. There are $default-mobile and $default-desktop, which set max and min widths, respectively. And there are $small, $medium and $large default breakpoints for mobile-first web design with multiple layouts.

Some modules also have a unique breakpoint variable included at the top of the file, which is set to $default-desktop or $default-mobile. You can change the module's breakpoint by giving that variable a new value.

      
        // Default Breakpoints
        // ===================
        // These are fairly meaningless default settings.
        // Feel free to reset these variables
        // to fit the needs of your specific project.

        $default-desktop: 768px;
        $default-mobile: 480px;

Media Queries

This guide includes a very simple, stripped down @mixin media() for basic media queries. It requires one variable â€” a breakpoint value - and has a secondary optional value $type, which defaults to 'min-width'.

      
        // Media Query Mixin
        // =========================

        @mixin media($bp, $type:'min-width') {
          @media ($type: #{$bp}) {
            @content;
          }
        }

        // Usage
        // Ex. 1
        @include media($default-desktop) {
          // any styles you want
          width: 100%;
        }

        // Ex. 2
        @include media($default-mobile, 'max-width') {
          // any styles you want
          width: 100%;
        }

Hide Mixin

Sometimes you'll want to hide an element on Desktop or Mobile. We've created a mixins to help:

If you only provide a $break-point (example: @include hide($default-desktop); ), the mixin will apply styles at that min-width. You can set a secondary variable to change the @mixin to look for max-width (example: @include hide($default-mobile, max-width);)

      
        // Hide Mixin
        // =========================

        @mixin hide($bp, $type:'min-width') {
          @media($type: $bp) {
            display: none;
          }
        }

Content-Block

Content-blocks are used to ensure that an element does not bump against the edge of the screen on smaller devices or window-sizes. Add the class .content-block to add a 10px left/right margin at all times, or add "@extend %content-block" to the parent-most element's styles. 

      
        %content-block,
        .content-block {
          @include pad(0 10px);
        }

Clear-List-Styles

If you want to use a <ul> as a building block without any added styles, you can remove the list-style-type/padding/margin with this class or @extend %clear-list-styles on another class.

      
        %clear-list-styles,
        .clear-list-styles {
          list-style-type: none;
          margin: 0;
          padding: 0;
        }
